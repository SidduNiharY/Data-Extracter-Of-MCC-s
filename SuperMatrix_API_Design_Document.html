Section 1

Product Overview

SuperMatrix Data Extractor is an internal web application built to consolidate campaign performance data from Google Ads (and other advertising platforms) into a single, unified reporting dashboard for internal use by our agency team.

Our agency manages Google Ads accounts for multiple clients across different verticals — e-commerce, lead generation, and service-based businesses. All client accounts are linked under a single Google Ads Manager Account (MCC ID: 844-978-1254). Today, account managers must log into each client account individually to pull reports — a process that is time-consuming and error-prone.

SuperMatrix solves this by using the Google Ads API to auto-discover all child accounts under the MCC with a single credential set, pull standardised campaign metrics on a scheduled basis, store them in a local PostgreSQL database, and present them in a single-pane dashboard for the internal team.

Key point: This tool is strictly read-only. It never creates, modifies, pauses, or deletes any campaigns, ad groups, ads, keywords, or bids. Its only interaction with the Google Ads API is fetching performance reports.

Core Objectives

  • Automatically discover all Google Ads client accounts linked to our MCC
  • Pull weekly campaign-level metrics per client account
  • Pull granular breakdowns: search terms, keywords, time segments, demographics
  • Store data locally for trend analysis and week-over-week comparisons
  • Give internal account managers a single dashboard instead of logging into each account

What the Tool Does NOT Do

  • Does not create, edit, pause, or delete any campaigns, ad groups, or ads
  • Does not modify bids, budgets, or targeting settings
  • Does not manage billing or payment information
  • Does not perform conversion tracking or remarketing
  • Does not expose client data to third parties
  • Is not accessible to clients or any external users

Section 3

Authentication & MCC Design

OAuth 2.0 Flow

SuperMatrix authenticates with the Google Ads API using a single OAuth 2.0 offline refresh token obtained from a Google account that is an administrator of the MCC (Manager Account ID: 844-978-1254). Credentials are stored in environment variables on the backend server and are never exposed to the frontend or any external party.

Credential Storage

  • GOOGLE_ADS_DEVELOPER_TOKEN — stored as environment variable
  • GOOGLE_ADS_CLIENT_ID — stored as environment variable
  • GOOGLE_ADS_CLIENT_SECRET — stored as environment variable
  • GOOGLE_ADS_REFRESH_TOKEN — stored as environment variable
  • GOOGLE_ADS_MCC_ID — stored as environment variable
Security: Credentials are never committed to version control. The .env file is excluded from git via .gitignore. Only the backend process can read these values.

MCC Account Discovery

All Google Ads API requests use login_customer_id set to the MCC ID. To discover child accounts, the tool runs the following GAQL query against the MCC:

SELECT customer_client.client_customer, customer_client.descriptive_name, customer_client.id, customer_client.manager, customer_client.status FROM customer_client WHERE customer_client.level <= 1

Results are filtered to exclude manager accounts (customer_client.manager = false). This gives a complete, accurate list of all direct child accounts under the MCC without querying any accounts outside of it.

Per-Account Data Pulls

Once child accounts are discovered, data is pulled for each by setting customer_id to the child account ID while keeping login_customer_id as the MCC. This is the standard pattern recommended by Google for MCC-based tools.

Section 4

API Usage & GAQL Queries

Read-only: All API interactions use GoogleAdsService.Search or GoogleAdsService.SearchStream. No mutate services are used. The tool never calls CampaignService.MutateCampaigns or any other mutate method.

Google Ads API Resources Used

Resource Purpose Campaign Types
customer_client Discover all child accounts under MCC All
campaign Campaign-level metrics: impressions, clicks, spend, CTR, conversions, ROAS Search, Shopping, Display, Performance Max, Video
search_term_view Top 10 search terms by clicks (last 7 days) Search, Performance Max
keyword_view Top 10 keywords by impressions, quality scores Search
campaign + segments.day_of_week Day-of-week performance breakdown All
campaign + segments.hour Hour-of-day performance breakdown All
gender_view Gender & age demographic breakdown Display, Video
customer Account name lookup (fallback) All

Sample GAQL Queries

Campaign Performance (last 7 days)

SELECT campaign.id, campaign.name, metrics.impressions, metrics.clicks, metrics.cost_micros, metrics.ctr, metrics.average_cpc, metrics.conversions, metrics.conversions_value, metrics.cost_per_conversion, metrics.search_impression_share, segments.date FROM campaign WHERE segments.date DURING LAST_7_DAYS AND campaign.status = 'ENABLED' ORDER BY metrics.impressions DESC

Top Search Terms (last 7 days, top 10 by clicks)

SELECT search_term_view.search_term, metrics.impressions, metrics.clicks, metrics.ctr, metrics.average_cpc, metrics.conversions FROM search_term_view WHERE segments.date DURING LAST_7_DAYS ORDER BY metrics.clicks DESC LIMIT 10

Demographics (gender & age)

SELECT ad_group_criterion.gender.type, ad_group_criterion.age_range.type, metrics.impressions, metrics.clicks, metrics.cost_micros, metrics.conversions, segments.date FROM gender_view WHERE segments.date DURING LAST_7_DAYS

Section 5

Data Flow & Storage

Data Pull Workflow

  1. A scheduled job (weekly, every Monday) or a manual trigger fires in the backend
  2. The backend queries the MCC via customer_client GAQL to get all child account IDs
  3. For each child account, the backend runs 6 GAQL read queries (campaign, search terms, keywords, time segments ×2, demographics)
  4. API responses are parsed, typed, and inserted into the local PostgreSQL database
  5. Each insert is deduplicated by (client_id, report_date) to prevent duplicate rows
  6. The pull job status (success/failed/partial) is logged to the pull_jobs table
  7. Internal users view the aggregated results in the Next.js dashboard

Data Security & Privacy

  • All data is stored on a private, internal server — not in any public cloud storage
  • The database is not accessible from outside the internal network
  • No client campaign data is shared with third parties or other tools
  • No personal user data (consumer PII) is collected — only aggregated ad performance metrics
  • API credentials are stored in environment variables, never in code or logs
Section 6

User Access & Permissions

Who Uses This Tool

SuperMatrix is an internal-only tool. Access is restricted to employees and contractors of our agency. Clients do not have access to this tool and cannot view it or log into it.

User TypeAccess LevelWhat They Can Do

Admin / Developer (internal) Full application access Configure clients, trigger manual pulls, manage MCC imports
External clients No access Not applicable — clients do not have login credentials
General public No access Application is not publicly accessible

No Third-Party API Usage

The Google Ads developer token is used exclusively with this internal tool. It is not shared with, resold to, or used in any tool developed by a third party. The same token is not used for any other application or service.

Summary: Internal employees only. No public access. No client access. No third-party usage of the developer token.

Compliance

  • The tool complies with Google Ads API Terms of Service
  • All API usage is within the rate limits appropriate for Basic Access (15,000 ops/day)
  • Credentials are rotated as needed and never shared externally
  • The API contact email is monitored and will respond to any notices from the Google Ads API team within 48 hours
